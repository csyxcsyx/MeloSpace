# 步骤一页面清单与流程基线

## 1. 角色

|角色|说明|核心能力|
|-|-|-|
|游客|未登录访问者|浏览首页、搜索、查看公开歌曲和公开歌单；需要登录后才能收藏、评论和建歌单。|
|普通用户|已登录用户|播放、收藏、评论、创建歌单、添加歌曲、查看最近播放和个人资料。|
|管理员|后台管理用户|管理歌曲、歌手、专辑、上传媒体、上下架歌曲、管理评论和用户状态。|

## 2. 页面清单

|页面|建议路由|优先级|主要内容|
|-|-|-|-|
|首页 / 发现页|`/`|P0|推荐区域、热门歌曲、新歌、场景入口、全局播放器入口。|
|登录 / 注册页|`/login`、`/register`|P0|账号密码登录、注册、错误提示、登录态保存。|
|搜索页|`/search`|P0|按歌曲、歌手、专辑、歌单综合搜索。|
|歌曲详情页|`/songs/:id`|P0|封面、歌曲信息、播放、收藏、加入歌单、歌词、评论。|
|歌单详情页|`/playlists/:id`|P0|歌单信息、歌曲列表、播放全部、添加/移除歌曲。|
|个人中心|`/me`|P1|个人资料、我的歌单、收藏、最近播放。|
|后台管理页|`/admin`|P0|歌曲管理、上传入口、上下架、歌手/专辑管理。|
|评论管理页|`/admin/comments`|P1|评论列表、删除或隐藏评论。|
|用户管理页|`/admin/users`|P1|用户状态管理、角色查看。|
|沉浸播放页|`/player`|P2|大封面、歌词高亮、播放队列、相关推荐。|
|场景推荐页|`/scenes`|P2|学习、深夜、运动、放松等规则推荐入口。|

全局播放器固定在应用底部，页面切换时保持当前歌曲、队列、播放模式、音量和进度。

## 3. 普通用户主流程

```text
打开首页
  -> 搜索或点击推荐歌曲
  -> 进入歌曲详情
  -> 点击播放
  -> 收藏歌曲
  -> 创建歌单
  -> 将歌曲加入歌单
  -> 发布评论
  -> 查看个人中心中的收藏、歌单和最近播放
```

成功标准：

- 用户 3 次点击内可以开始播放歌曲。
- 播放失败时页面不崩溃，并显示明确错误。
- 收藏、歌单、评论和最近播放能持久化。
- 未登录用户执行收藏、评论、建歌单时跳转或提示登录。

## 4. 管理员主流程

```text
管理员登录
  -> 进入后台管理页
  -> 上传音频、封面、歌词
  -> 填写歌曲、歌手、专辑、标签、情绪信息
  -> 上架歌曲
  -> 回到前台搜索该歌曲
  -> 播放验证
  -> 必要时下架或编辑歌曲
```

成功标准：

- 普通用户不能访问后台接口或后台页面。
- 上传文件有大小、类型和扩展名限制。
- 歌曲上架后前台可搜索、详情可查看、播放器可播放。
- 歌曲下架后不出现在普通用户可播放列表中。

## 5. 答辩演示主线

1. 展示首页，说明项目基于教材音乐网并升级为现代音乐平台体验。
2. 登录普通用户，展示个人中心、收藏、最近播放入口。
3. 搜索并播放歌曲，展示底部全局播放器。
4. 打开歌曲详情，展示歌词和评论。
5. 创建歌单并添加歌曲。
6. 展示两个交互亮点：优先歌词高亮、拖拽歌单或场景推荐。
7. 登录管理员，上传并上架歌曲。
8. 展示 Docker Compose / Nginx / MySQL 持久化部署方案。

## 6. 页面验收点

- 首页、歌曲详情、歌单详情、后台页风格统一。
- 所有列表页面具备加载态、空状态和错误态。
- 表单具备基础校验和防重复提交状态。
- 桌面端、笔记本端和移动端核心流程可用。
- 全局播放器始终可见或在移动端有明确入口。
